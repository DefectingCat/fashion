# Markdown 编辑器图片粘贴上传功能分析报告

## 一、功能现状分析

### 1.1 当前编辑器实现

- 使用 `@uiw/react-md-editor` v4.1.0
- 支持基本的 Markdown 编辑和实时预览
- **不原生支持**图片粘贴自动上传功能

### 1.2 后端上传接口

**接口路径**: `POST /api/upload`

- **认证**: Bearer Token (JWT)
- **请求格式**: FormData (`file` 字段)
- **支持格式**: jpeg, png, gif, webp
- **文件大小限制**: 最大 5MB
- **存储位置**: `public/uploads/`
- **返回格式**: `{ success: true, url: "/public/uploads/{filename}" }`

## 二、功能实现方案

### 2.1 技术架构

```
┌─────────────────┐    onPaste    ┌─────────────────────┐
│   MDEditor      ├──────────────►│ PasteImageUpload    │
│   (编辑器)      │               │ (HOC 组件)         │
└─────────────────┘               └──────────┬──────────┘
                                              │
                                              ▼
                                    ┌─────────────────┐
                                    │  POST /api/upload │
                                    │  (后端接口)      │
                                    └────────┬────────┘
                                             │
                                             ▼
                                    ┌─────────────────┐
                                    │  insert Markdown │
                                    │  ![alt](url)    │
                                    └─────────────────┘
```

### 2.2 核心组件

#### 1. `PasteImageUpload.tsx` (HOC 组件)

- 拦截编辑器的 `onPaste` 事件
- 检测剪贴板中的图片文件
- 调用 `/api/upload` 上传图片
- 成功后插入 Markdown 图片语法

#### 2. `PostEditor.tsx` 集成

- 添加 `imageUploading` 状态显示上传进度
- 集成 `PasteImageUpload` 组件包裹 MDEditor
- 提供上传开始/结束/错误回调

## 三、功能测试步骤

### 3.1 前置条件

1. 已登录系统 (使用 <admin@example.com> / admin123)
2. 访问 `/admin/posts/new` 或编辑已有文章

### 3.2 测试用例

| 测试项       | 操作步骤         | 预期结果                         |
| --------- | ------------ | ---------------------------- |
| **粘贴图片**  | 复制图片后在编辑器中粘贴 | 图片自动上传，插入 `![filename](url)` |
| **粘贴文本**  | 复制普通文本后粘贴    | 正常粘贴文本，不上传                   |
| **大文件**   | 粘贴 >5MB 的图片  | 显示"文件太大"错误                   |
| **非图片文件** | 粘贴非图片文件      | 正常粘贴，不上传                     |
| **上传进度**  | 粘贴图片时        | 显示"正在上传图片..."加载动画            |
| **上传失败**  | 网络异常时粘贴      | 显示错误提示                       |

### 3.3 验证方法

1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签
3. 粘贴图片后查看是否有 `POST /api/upload` 请求
4. 验证请求响应是否包含正确的 URL
5. 检查编辑器是否插入了正确的 Markdown 语法

#### 4. 上传重试机制

```tsx
const uploadWithRetry = async (file: File, retries = 3) => {
  for (let i = 0; i < retries; i++) {
    try {
      return await uploadImage(file)
    } catch (err) {
      if (i === retries - 1) throw err
      await sleep(1000 * (i + 1))
    }
  }
}
```

### 5.2 `document.execCommand` 说明

当前使用 `document.execCommand('insertText')` 插入文本，虽然该 API 已被标记为过时，但在所有主流浏览器中仍有良好支持。

**替代方案** (如需更现代的实现):

```tsx
// 使用 MDEditor 的 API
// 需要获取编辑器实例并调用 replaceSelection
```

### 5.3 跨域问题

当前架构**不存在跨域问题**，因为：

- 前端和后端在同一域名 (`localhost:3000`)
- API 请求是同源请求

如果需要分离部署，需要在后端配置 CORS：

```typescript
// 在 Elysia 中添加 CORS 支持
import { cors } from '@elysiajs/cors'

app.use(cors())
```

## 六、使用说明

### 6.1 用户操作流程

1. 登录管理后台
2. 创建或编辑文章
3. 复制图片 (截图或从其他地方复制)
4. 在编辑器中直接粘贴 (Ctrl+V / Cmd+V)
5. 等待上传完成，图片将自动插入

### 6.2 支持的图片来源

- ✅ 截图工具 (如 Snipaste, 系统截图)
- ✅ 网页图片复制
- ✅ 本地图片文件复制
- ✅ 图片拖拽 (待扩展)
