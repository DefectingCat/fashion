## 1. 总则

### 1.1 注释原则

- **必要性原则**：注释应该解释"为什么"而不是"是什么"，代码本身应该能说明"是什么"
- **准确性原则**：注释必须与代码保持一致，代码修改时同步更新注释
- **简洁性原则**：注释应简洁明了，避免冗长的废话
- **中文优先**：项目注释使用中文，面向国内开发者

### 1.2 注释语言

- 所有注释使用**简体中文**
- 专业术语可保留英文（如：React, TypeScript, API）
- 中英文混合时，英文前后各加一个空格

### 1.3 禁止事项

- ❌ 禁止注释掉的代码（应直接删除）
- ❌ 禁止无意义的注释（如：`// i++ 增加 i`）
- ❌ 禁止与代码矛盾的注释
- ❌ 禁止使用错别字和语法错误

## 2. 文件头注释规范

### 2.1 基本格式

每个文件顶部必须包含文件头注释，格式如下：

```typescript
/**
 * @file 文件功能描述
 * @description 详细的文件功能说明（可选，简单文件可省略）
 * @author 作者姓名 <作者邮箱>
 * @created 创建日期（格式：YYYY-MM-DD）
 * @lastModified 最后修改日期（格式：YYYY-MM-DD，可选）
 * @copyright 版权信息（可选）
 */
```

### 2.2 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `@file` | ✅ | 文件功能简短描述 | 用户认证相关路由 |
| `@description` | ❌ | 详细功能说明 | 提供登录、注册、JWT 验证等功能 |
| `@author` | ✅ | 作者信息 | 张三 <zhangsan@example.com> |
| `@created` | ✅ | 创建日期 | 2024-01-15 |
| `@lastModified` | ❌ | 最后修改日期 | 2024-02-20 |
| `@copyright` | ❌ | 版权信息 | Copyright (c) 2024 Fashion Blog |

### 2.3 示例

#### 示例 1：简单文件

```typescript
/**
 * @file 文章管理相关 API 路由
 * @author 张三 <zhangsan@example.com>
 * @created 2024-01-15
 */
```

#### 示例 2：复杂文件

```typescript
/**
 * @file Markdown 编辑器图片粘贴上传组件
 * @description 拦截编辑器的粘贴事件，自动检测剪贴板中的图片并上传至服务器
 * @author 李四 <lisi@example.com>
 * @created 2024-02-10
 * @lastModified 2024-02-15
 * @copyright Copyright (c) 2024 Fashion Blog
 */
```

## 3. 函数注释规范

### 3.1 JSDoc 格式

所有函数（包括箭头函数）必须使用 JSDoc 格式注释，放置在函数定义上方。

```typescript
/**
 * 函数功能概述
 *
 * 详细描述函数的作用、实现逻辑、使用场景等（可选）
 *
 * @param 参数名 - 参数描述
 * @param 参数名 - 参数描述，支持多行
 * @returns 返回值描述
 * @throws 异常类型 - 异常描述
 * @example 使用示例（可选）
 */
```

### 3.2 函数注释字段

| 标签 | 必填 | 说明 |
|------|------|------|
| 功能概述 | ✅ | 一句话说明函数做什么 |
| `@param` | 有参数时必填 | 每个参数都需要单独的 @param 标签 |
| `@returns` | 有返回值时必填 | 说明返回值的含义和类型 |
| `@throws` | 可能抛出异常时必填 | 说明可能抛出的异常类型和原因 |
| `@example` | ❌ | 提供使用示例 |
| `@deprecated` | ❌ | 标记已废弃的函数 |

## 6. 行内注释规范

### 6.1 注释位置

行内注释应放在被解释代码的**上方**或**右侧**。

#### 6.1.1 上方注释（推荐）

```typescript
// 检查用户是否有编辑权限
if (user.role === 'admin' || post.author_id === user.id) {
  // 允许编辑
}
```

#### 6.1.2 右侧注释（简短注释）

```typescript
const maxFileSize = 5 * 1024 * 1024; // 最大 5MB
const validTypes = ['image/jpeg', 'image/png', 'image/gif', 'image/webp']; // 支持的图片格式
```

### 6.2 注释内容要求

- ✅ 解释**为什么**这样做，而不是**怎么做**
- ✅ 解释复杂的业务逻辑
- ✅ 解释非常规的实现方式
- ✅ 标记待办事项和临时方案

### 6.3 行内注释示例

#### 示例 1：解释业务逻辑

```typescript
// 使用 bcrypt 哈希密码，盐值 rounds 设为 10
// 越高越安全但越慢，10 是安全性和性能的平衡点
const hashedPassword = await bcrypt.hash(password, 10);
```

#### 示例 2：解释复杂算法

```typescript
// 计算分页偏移量
// 公式：offset = (page - 1) * pageSize
// 例如：第 3 页，每页 10 条，偏移量为 20
const offset = (page - 1) * pageSize;
```

#### 示例 3：标记临时方案

```typescript

// 当前使用内存缓存，重启后数据会丢失

## 8. 注释示例

### 8.1 完整文件示例

```typescript
/**
 * @file 图片粘贴上传组件
 * @description 拦截编辑器的粘贴事件，自动检测剪贴板中的图片并上传至服务器
 * @author 李四 <lisi@example.com>
 * @created 2024-02-10
 */

import React from 'react';

interface PasteImageUploadProps {
  /** JWT 认证 token */
  token: string | null;
  /** 上传开始回调 */
  onUploadStart?: () => void;
  /** 上传结束回调 */
  onUploadEnd?: () => void;
  /** 上传错误回调 */
  onError?: (error: string) => void;
  /** 子组件（MDEditor） */
  children: React.ReactElement;
}

/**
 * 图片粘贴上传高阶组件
 *
 * 包裹 Markdown 编辑器，拦截粘贴事件，自动上传剪贴板中的图片
 *
 * @param props - 组件属性
 * @param props.token - JWT 认证 token
 * @param props.onUploadStart - 上传开始回调
 * @param props.onUploadEnd - 上传结束回调
 * @param props.onError - 上传错误回调
 * @param props.children - 子组件（MDEditor）
 * @returns 包装后的编辑器组件
 * @example
 * ```tsx
 * <PasteImageUpload token={token}>
 *   <MDEditor value={content} onChange={setContent} />
 * </PasteImageUpload>
 * ```
 */
export function PasteImageUpload({
  token,
  onUploadStart,
  onUploadEnd,
  onError,
  children,
}: PasteImageUploadProps) {
  /**
   * 处理粘贴事件
   *
   * @param event - 剪贴板事件
   */
  const handlePaste = async (event: React.ClipboardEvent) => {
    const clipboardData = event.clipboardData;

    // 检查剪贴板中是否有文件
    if (!clipboardData || clipboardData.files.length === 0) {
      return;
    }

    const file = clipboardData.files[0];

    // 只处理图片文件
    if (!file.type.startsWith('image/')) {
      return;
    }

    // 阻止默认粘贴行为
    event.preventDefault();
    onUploadStart?.();

    try {
      // 构建上传表单数据
      const formData = new FormData();
      formData.append('file', file);

      // 调用上传接口
      const res = await fetch('/api/upload', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${token}`,
        },
        body: formData,
      });

      if (!res.ok) {
        throw new Error('图片上传失败');
      }

      const data = await res.json();
      // 生成 Markdown 图片语法
      const imageMarkdown = `![${file.name}](${data.url})\n`;

      // HACK: 使用 execCommand 插入文本
      // 该 API 已过时，但目前是最兼容的方式
      document.execCommand('insertText', false, imageMarkdown);
    } catch (err) {
      const errorMsg = err instanceof Error ? err.message : '图片上传失败';
      onError?.(errorMsg);
    } finally {
      onUploadEnd?.();
    }
  };

  // 克隆子组件并添加 onPaste 事件
  const childWithProps = React.cloneElement(children, {
    onPaste: handlePaste,
  });

  return childWithProps;
}
```
